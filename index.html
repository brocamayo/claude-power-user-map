Claude Cheat Sheet

A quick reference for getting better results with Claude.

General Developers
Pick Your Tool
⚡ Claude Code
Multi-file code edits, shell access, git. For real software builds.
📁 Projects
Persistent context across chats. For recurring tasks.
💬 Chat
Quick and flexible. For one-off questions, drafts, ideas.
🤖 API
Build AI into your products. Custom agents & pipelines.
The Workflow
1. Load context → 2. Frame problem → 3. Ask for plan first → 4. Build in pieces → 5. Review & test → 6. Document & ship
⚠️ Never skip step 3. If a task touches 2+ files or takes 10+ min manually, always ask for a plan first.
Patterns Worth Knowing
🔗 Prompt Chaining Common
Break big tasks into steps. Each output feeds the next. Add checkpoints between.
🔄 Critic-Refine Quality
Generate → critique → improve. Use a fresh session for the critique (Claude is too nice to itself).
🛠 Eval-Fix Loop Code
Generate code → run tests → feed errors back → fix. Set a max of 3-5 loops.
🧰 Orchestrator Advanced
Give different Claude calls different roles (researcher, builder, reviewer). You pass outputs between them.

Context Rules
✅ Do
  • Paste full error messages
  • Include exact file paths
  • State language + versions
  • Say what you expected vs. what happened
⛔ Don't
  • Summarize errors in your own words
  • Dump huge files without pointing to the issue
  • Assume it remembers last session
  • Let context bloat past ~70% on code tasks
🔄 Start fresh when:
Responses drift or hallucinate · Claude contradicts itself · Quality drops · 30+ turns deep · It re-suggests things you rejected
8 Golden Rules
🎯 One job per prompt
Don't ask it to design + build + test + document in one shot.
📋 Plan before building
Ask for an outline first. Review it. Then say "proceed."
🔁 Never accept draft one
Generate → critique → refine. 3x better every time.
📏 Be specific about output
"Write a function" is weak. Add format, types, and constraints.
🧪 Write tests first
Forces clear acceptance criteria before any code is written.
🔐 Adversarial review
"Try to break this. What are the failure modes?"
📌 Repeat key constraints
Put critical rules at the top AND bottom of long prompts.
🗃 Use Projects for repeat work
Persistent context beats re-explaining yourself every time.

Know Your Tools
The Four Surfaces

Claude isn't one tool — it's four. Picking the right one for the job is the single biggest factor in your results.

⚡
Claude Code
The Power Tool
Multi-file editing, shell access, git integration. Use this for building real software — refactors, new features, full-stack apps.
  • Multi-file edits across a codebase
  • Runs shell commands & tests
  • Built-in plan mode for safety
📁
Projects
Persistent Memory
Upload docs and set a system prompt that stays active across every conversation. Your reusable knowledge base.
  • Context carries between chats
  • Great for recurring tasks
  • Upload style guides & docs
💬
Chat
Quick & Flexible
The fastest path for one-off tasks — brainstorming, drafts, analysis, learning. No setup, just go.
  • Zero setup overhead
  • Fast iteration on ideas
  • Best for exploration
🤖
API / Artifacts
Build With AI
Embed Claude in your own products and pipelines. Artifacts let you create interactive UIs right in chat.
  • Custom agents & integrations
  • Automated pipelines
  • AI-powered product features
💡
Quick rule: Need shell access? → Claude Code. Same task every week? → Projects. One-off question? → Chat. Building a product? → API.
How to Work
The 6-Step Workflow

Whether you're building an app, writing a doc, or analyzing data — this loop works. The key insight: always plan before you build.

1
Load Context
Give Claude everything it needs upfront — docs, schemas, error logs, examples. Don't summarize errors; paste them in full. The more context Claude has, the better the output.
2
Frame the Problem
State what you have, what you want, and what you've already tried. Give a concrete example of the desired output. Mention any constraints (language, dependencies, performance).
3
Ask for a Plan First
Before Claude writes anything, ask it to outline its approach. In Claude Code, use --plan mode. In chat, just say "outline your approach before writing." Review the plan, then approve.
4
Build Incrementally
Don't try to build everything at once. Work in small pieces — one module, one function, one section at a time. Test each piece before moving on.
5
Review & Stress-Test
Ask Claude to critique its own work: "What would break this? What edge cases am I missing? What are the security concerns?" A separate review prompt catches things the build prompt missed.
6
Document & Ship
Have Claude generate documentation, README, or handoff notes while the context is fresh. Log what was built and why — future-you will thank you.
⚠️
The #1 mistake: Skipping step 3. If a task touches more than 2 files or takes 10+ minutes manually, asking for a plan first will save you from expensive rewrites.
Level Up
Powerful Patterns

Once you've nailed the basics, these patterns let you chain multiple AI interactions together for much better results.

🔗
Prompt Chaining
Break big tasks into sequential steps
Intermediate
Instead of one massive prompt, break work into stages. Each step's output feeds the next. Add human checkpoints between stages.
Example:
Research → Outline → Draft → Edit → Ship
🔄
Critic-Refine Loop
Generate, critique, improve — repeat
Intermediate
Never accept the first draft. Ask Claude to generate, then critique its own work, then refine the weak parts. This simple loop produces dramatically better output.
⚠ Tip: Use a fresh conversation for the critique step — Claude tends to be too kind to its own work in the same session.
🛠
Eval-Fix Loop
Generate code, run tests, fix failures, repeat
Advanced
For code: have Claude generate it, run your tests, feed the errors back, and let it fix them. Set a max of 3–5 iterations so it doesn't loop forever.
Generate → Test → Feed errors → Fix → Pass? Ship!
🧰
Orchestrator Pattern
You design the pipeline, Claude runs each role
Advanced
For complex projects: break the work into roles (researcher, builder, reviewer, writer). Give each role its own focused prompt and pass outputs between them. You're the project manager.
Key insight: Multiple focused Claude calls with specific system prompts beat one massive do-everything prompt every time.
🚀
Start simple: Most tasks only need single-prompt or prompt chaining. Don't reach for loops and orchestration until simpler approaches fall short.
Stay Sharp
Context Hygiene

The quality of Claude's output is directly tied to the quality of context you provide. These rules keep conversations productive.

✅ Always Do
  • Paste full error messages — never summarize
  • Include exact file paths
  • State language, runtime, and library versions
  • Explain what you expected vs. what happened
  • Open long sessions with a context brief
⛔ Never Do
  • Summarize errors in your own words
  • Paste 1000+ lines without flagging the key area
  • Start a new session mid-problem without a recap
  • Assume Claude remembers previous sessions
  • Let context bloat past 70-80% on precision tasks
🔄 When to Start a Fresh Session
Responses start drifting or hallucinating
Claude contradicts something it said earlier
Output quality drops noticeably
You're 30+ turns deep on one problem
It suggests something you already ruled out
It stops following your constraints
Context Window Health
0-60% — Optimal 60-80% — Watch for drift 80%+ — Consider a fresh session
Choose Wisely
Which Model to Use

Bigger isn't always better. Matching the right model to the task saves money, reduces wait time, and often produces better results.

★
Opus
Maximum Brainpower
The most capable model. Use for tasks where quality and depth matter most.
  • Complex architecture decisions
  • Nuanced writing & strategy
  • Hard debugging & edge cases
  • Security audits
Slowest Highest quality
◆
Sonnet
The Sweet Spot
Handles 80% of tasks at near-Opus quality. Start here unless you have a specific reason not to.
  • Day-to-day coding
  • Content drafting & editing
  • Data analysis
  • Agentic workflows
Balanced Best default
△
Haiku
Fast & Cheap
Best for simple, well-defined tasks — especially at high volume where speed and cost matter.
  • Classification & tagging
  • Data extraction
  • Simple Q&A
  • Routing in pipelines
Fastest Lowest cost

Writing features or fixing bugsSonnetGreat quality at reasonable speed

Cascade trick: Use Haiku to triage, Sonnet to execute, Opus only for final review. Cuts costs 5-10x.
⚠️
Don't default to Opus. Sonnet handles most tasks at comparable quality. Save Opus for the 20% that genuinely need maximum depth.
🌐
Works across providers: This tiering applies everywhere. GPT-4o ≈ Sonnet, GPT-4o-mini ≈ Haiku, o1/o3 ≈ Opus-class. Gemini Pro/Flash follow the same split.
Quick Reference
8 Rules That Make Everything Better

Pin these to your wall. They're model-agnostic — they work with any AI tool.

🎯
One Job Per Prompt
Don't ask Claude to design, build, test, AND document in one message. Break it up.
📋
Plan Before You Build
Always ask Claude to outline its approach before writing code or content. Review, then approve.
🔁
Iterate, Don't Accept
Generate, critique, refine is 3x better than taking the first answer. Push for improvement.
📏
Be Specific About Output
"Write a function" is weak. "Write a Python function that returns X given Y, with type hints" is strong.
🧪
Tests Before Code
Ask Claude to write tests first. It forces clear thinking about what success looks like.
🔐
Adversarial Review
"Act as a senior engineer trying to break this. What are the failure modes?" catches what you'd miss.
📌
Repeat Key Constraints
Put your most important rules at the beginning AND end of long prompts. Attention fades in the middle.
🗃
Use Projects for Repeat Work
If you do the same kind of task weekly, set up a Project. Persistent context beats re-explaining every time.
